# UX_UI_DIRECTION

## Desired user experience
Quiet, fast, trustworthy, operational. The product should feel like a serious workbench for recurring case work, not like a marketing dashboard.

## Desired visual style
- Dense but readable layouts.
- Clear tables and split-pane case views.
- Subtle color coding for risk/status.
- Minimal decorative elements.
- Strong focus on scanability.

## Reference product patterns
- Linear-style issue clarity.
- Notion-style structured notes, but more operational.
- Intercom/Zendesk-style case queues.
- Modern admin dashboards with restrained visual design.

## Information hierarchy
Global:
- Workspace switcher.
- Cases.
- Dashboard.
- Templates.
- Team/admin.

Case detail:
- Case title/status/owner/risk at top.
- Current summary and next action above the fold.
- Tabs or sections: Overview, Tasks, Notes, Files, Activity, AI.

## Most important screens
- Login/signup.
- Case dashboard/list.
- Case detail.
- Create case.
- AI summary review panel.
- Admin/team settings.

## Navigation expectations
- Sidebar for primary app areas.
- Table/list for case portfolio.
- Case detail should support quick back to list.
- Filters should be persistent during a session.

## Mobile-first / desktop-first / both
Desktop-first for MVP. Mobile should be readable for review/checking status, but full case work can be desktop-optimized.

## Accessibility expectations
- Keyboard-accessible core actions.
- Clear focus states.
- Sufficient color contrast.
- Status must not rely on color alone.

## Things to avoid
- Landing-page style hero screens inside the app.
- Overly playful UI.
- Hidden critical actions.
- AI outputs that look indistinguishable from verified human facts.
