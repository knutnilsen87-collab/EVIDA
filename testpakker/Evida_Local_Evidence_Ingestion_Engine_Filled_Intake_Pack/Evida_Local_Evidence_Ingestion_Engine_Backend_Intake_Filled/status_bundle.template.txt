status_bundle: Evida Local Evidence Ingestion Engine

facts:
- Evida is a local desktop application.
- The ingestion engine must run locally inside Evida.
- The engine must be architecturally separate and reusable.
- Documentation imported into Evida may be decisive for a case.
- All discovered files/pages must receive explicit status.
- Users need live counts, ETA, remaining documents/pages and readiness.
- Users need a simple manual review queue for unreadable/problem files/pages.
- AI must distinguish partial usable material from complete verified case material.
- Manual review does not equal machine-readable OCR/text extraction.

assumptions:
- SQLite WAL is acceptable for local persistence.
- Local-only is default.
- OCR can be local in MVP or implemented behind adapter.
- Evida has or will have a desktop UI capable of showing import dashboard and previews.
- Exact desktop framework is not yet locked.
