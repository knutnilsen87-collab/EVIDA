## Minimumsoppløsning

Bruk helst desktopstørrelse som matcher appens faktiske releasevindu.

Typisk:

```text
1600x900 eller større
```

## Hva skjermbildene skal vise

- hvordan appen ser ut rett etter oppstart
- at det ikke finnes login i lokal evaluation-build
- hvordan dokumentimportens status og ETA ser ut
- hvordan feil/manuell kontroll vises
- hvordan Saksrom svarer med kilder
- hvordan drawers/modaler ser ut
- hvordan analyseflater og eksport er strukturert

## Ikke ta med

- reelle klientnavn
- reelle dokumenter
- personopplysninger
- sensitiv saksinformasjon

Bruk bare syntetiske testpakker fra repoets testdata.
