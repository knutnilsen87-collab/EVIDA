# MASTER_DELIVERY_ROADMAP

## Product target
Evida should become a production-grade AI-assisted SaaS workspace for case management.

## Phase 4: MVP implementation plan and backlog
Goal: convert product direction into implementation-ready artifacts.

Exit criteria:
- Database schema defined.
- API route map defined.
- UI screen map defined.
- MVP backlog prioritized.
- Test plan defined.
- Local dev setup defined.

## Phase 5: Build MVP
Goal: implement the first usable product.

Exit criteria:
- Auth and workspace membership work.
- Users can create, update, archive, and view cases.
- Users can manage notes, tasks, and file metadata.
- AI summary draft and review flow works.
- Dashboard/list filters work.
- Permission checks exist server-side.
- MVP test suite passes.

## Phase 6: AI and evaluation hardening
Goal: make AI useful, traceable, bounded, and safe for pilot users.

Exit criteria:
- Prompt versions tracked.
- AI outputs separated from human facts.
- Evaluation dataset exists.
- Guardrails and failure states exist.
- Cost/latency logging exists.
- Human approval flow is enforced.

## Phase 7: Production hardening and beta
Goal: make the product safe for real pilot usage.

Exit criteria:
- Production environment configured.
- Monitoring/error tracking enabled.
- Backups and restore path verified.
- Security review completed.
- Beta onboarding, support, and feedback process exists.
- No known P0/P1 issues.

## Phase 8: Launch
Goal: release Evida as a controlled production product.

Exit criteria:
- Domain and production deploy ready.
- Legal/privacy pages ready.
- Launch checklist complete.
- Support process ready.
- Rollback plan ready.
- First users onboarded.

## Recommended sequencing
Do not begin Phase 8 before Phase 7 exit criteria are true. Do not begin broad AI automation before Phase 6 exit criteria are true.
