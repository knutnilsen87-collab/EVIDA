# MVP_RELEASE_SCOPE

## Included in first MVP release
- Auth.
- Workspace membership.
- Roles.
- Case list.
- Case detail.
- Case create/edit/archive.
- Notes.
- Tasks.
- File metadata and private uploads.
- Activity timeline.
- AI summary draft/review.
- Dashboard basics.
- Audit log basics.
- Admin member management.

## Excluded from first MVP release
- Billing.
- External client portal.
- Email ingestion.
- Calendar integration.
- Document RAG.
- Custom workflow builder.
- Advanced analytics.
- Public API.
- Mobile app.

## Release assumption
The first release is a controlled pilot, not a broad public launch.

## Pilot user requirements
Pilot users should understand:
- AI output is advisory.
- Case summaries must be reviewed.
- Sensitive uploads should follow agreed data policy.
- Feedback and bug reports are expected during beta.

## Success metrics
- 20+ real cases created by pilot users.
- 80%+ of active cases have owner, status, risk, and next action.
- AI summary used on at least 50% of active cases.
- No critical permission/security incidents.
- Users report less need for separate status spreadsheets.
