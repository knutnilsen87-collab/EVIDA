# Implemented vs Planned

| Area | Implemented now | Planned / not done |
| --- | --- | --- |
| Desktop shell | Tauri app, React UI, build passes | Signed release and clean-machine verification |
| Saksrom | Chat-first UI, summary, source references, sticky composer, adaptive suggestions | Stronger answer evaluation and production policy gates |
| Import/processing | Local import UX, progress states, ETA helpers, test coverage | Production-grade OCR/storage/security model |
| Legal workrooms | Chronology, evidence, arguments, contradictions, risk, simulation views exist | Deeper legal validation and export-quality workflow |
| Backend ownership | Canonical docs say Spring Boot owns enterprise/control plane | Full integration and FastAPI deprecation cleanup |
| Security | Security policy docs and boundary verifier | Auth, tenant isolation, encryption, audit chain, provider enforcement |
| Release | Build and Tauri bundle commands work locally | Release signing, SBOM, installer QA, public vulnerability intake |
| Documentation | Rich phase docs and ADRs | One maintained source-of-truth index and stale-doc pruning |

Bottom line: there is a credible local evaluation app. There is not yet a production legal platform.
