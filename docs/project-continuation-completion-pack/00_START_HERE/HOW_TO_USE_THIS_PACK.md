# How To Use This Pack

Use this folder as the project truth snapshot before starting another implementation pass.

Recommended flow:
1. Read `01_PROJECT_IDENTITY` to understand what Evida is and is not.
2. Read `02_REALITY_CHECK` before making roadmap claims.
3. Use `03_SOURCE_OF_TRUTH` to decide which docs/code win when artifacts conflict.
4. Use `04_PRODUCT_SCOPE_STATUS` to avoid expanding scope before MVP hardening.
5. Use `05_TECHNICAL_BASELINE` before changing architecture or dependencies.
6. Use `06_GAPS_BLOCKERS_TECH_DEBT` for risk-based prioritization.
7. Execute `07_COMPLETION_PLAN/FIRST_10_FIXES.md` before broader roadmap work.
8. Paste `09_STATUS_BUNDLE_CONTINUATION/status_bundle.template.txt` into the next continuation thread.

Update this pack after material changes, especially after releases, backend ownership changes, security work, or feature completion.
