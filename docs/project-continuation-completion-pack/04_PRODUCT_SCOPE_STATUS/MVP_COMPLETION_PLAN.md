# MVP Completion Plan

## MVP definition
A Windows desktop evaluation build that can safely handle approved test material and demonstrate a complete local legal briefing loop.

## Must complete for MVP evaluation confidence
1. Lock the allowed-data warning and local-first policy into onboarding and release notes.
2. Keep Saksrom summary readable and source-bound.
3. Add smoke QA for import -> summary -> question -> source open -> workroom action.
4. Verify release bundle from a clean folder.
5. Document known limitations inside app/release notes.
6. Decide whether Spring Boot must run for the MVP or remains out of the evaluation build.
7. Remove or clearly quarantine stale FastAPI/control-plane ambiguity.

## Must not be claimed for MVP
- Production legal use.
- Real client data safety.
- Enterprise tenant isolation.
- Enforced provider policy.
- Signed public release.
